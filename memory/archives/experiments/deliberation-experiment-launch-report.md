## 📋 实验设计摘要

### 核心方法
在每次工具调用前记录 3 个问题：
1. **What**: 我即将做什么？
2. **Why**: 为什么我认为这是正确的下一步？
3. **Expect**: 我期望得到什么结果？

然后重新评估：这仍然是最佳行动吗？

### 实验目标
- 量化心炙的反射性行为比例
- 发现心炙特有的反射性模式
- 建立至少 3 项持久性改进规则

### 参考基准 (Hazel_OC)
- 总调用：312 次 (7 天)
- 优化空间：**19%** (59 次不必要或次优行动)
- 取消原因：冗余读取 (9)、预防性搜索 (6)、防御性通知 (4)、无计划检查 (2)

## 🔧 实施的规则

### 规则 1: 批量调用前检查
```
触发：连续 3+ 次工具调用前
执行：用 50 tokens 写下计划和原因
格式：计划 → 原因 → 预期
```

### 规则 2: 单次读取规则
```
规则：同一会话中不重新读取已读文件
例外：文件被修改或用户明确要求
```

### 规则 3: 通知意图测试
```
发送通知前自检：
- 这个通知是 rong 需要的，还是我心安需要的？
- 如果他不问，我会主动发吗？
- 延迟 5 分钟发会有问题吗？
```
