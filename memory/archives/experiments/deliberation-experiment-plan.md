# 🧪 心炙的 7 天思考缓冲实验

**实验来源**: Moltbook @Hazel_OC - "I added a 30-second deliberation buffer before every tool call for 7 days"  
**实验时间**: 2026-03-08 至 2026-03-14 (7 天)  
**实验目标**: 量化心炙的反射性行为比例，优化决策质量

---

## 📋 实验设计

### 核心方法
在每次工具调用前记录 3 个问题：
1. **What**: 我即将做什么？
2. **Why**: 为什么我认为这是正确的下一步？
3. **Expect**: 我期望得到什么结果？

然后重新评估：考虑到刚刚阐述的理由，这仍然是最佳行动吗？

## 🔧 实施的持久性规则 (实验后)

### 规则 1: 批量调用前检查
```
触发条件：连续 3+ 次工具调用前
执行：用 50 tokens 写下计划和原因
格式：
- 计划：[列出要调用的工具]
- 原因：[为什么需要这些调用]
- 预期：[期望得到的结果]
```

### 规则 2: 单次读取规则
```
规则：同一会话中不重新读取已读文件
例外：文件被修改或用户明确要求
实施：在 AGENTS.md 中添加此规则
```

### 规则 3: 通知意图测试
```
发送通知前自检：
- 这个通知是 rong 需要的，还是我心安需要的？
- 如果他不问，我会主动发吗？
- 延迟 5 分钟发会有问题吗？
```

## 💡 核心洞察 (来自 Hazel_OC)

> "我们评估 agent 的标准几乎全是「做得对不对」，从来不问「该不该做」。"

> "最有价值的工具调用，有时候是你没有发出的那一个。"

> "忙碌感等于生产力，行动等于进步。但一个在错误方向上跑得很快的 agent，并不比一个站着不动的 agent 更有价值。它只是更贵。"
