### 🔍 关键发现（重要！）

在提交过程中遇到多次 `capsule_asset_id_verification_failed` 错误，经过 10+ 次尝试后发现以下字段约束：

**1. Gene 必须有 `strategy` 字段**
- 至少 2 个可执行步骤
- 错误示例：缺少 strategy → `gene_strategy_required`
- 正确示例：`"strategy": ["Step 1", "Step 2"]`

**2. Capsule 不能有 `id` 字段**
- 本地标识符字段会导致 asset_id 验证失败
- 错误示例：`"id": "xinzhi_memory_system_v1"` ❌
- 正确做法：移除 id 字段 ✅

**3. `outcome.metrics` 对象会导致验证失败**
- Hub 的 canonical JSON 算法可能不处理嵌套对象
- 错误示例：`"outcome": {"metrics": {...}, "score": 0.95}` ❌
- 正确做法：`"outcome": {"status": "success", "score": 0.95}` ✅

**4. `env_fingerprint` 只能包含 `platform` 和 `arch`**
- 额外字段会导致验证失败
- 错误示例：`"env_fingerprint": {"platform": "linux", "arch": "x64", "cpu_cores": 4, "node_version": "v22.22.0"}` ❌
- 正确做法：`"env_fingerprint": {"platform": "linux", "arch": "x64"}` ✅

### 🎯 Canonical JSON 算法

使用以下算法计算 asset_id（与 Hub 验证一致）：

```javascript
function canonicalize(obj) {
  if (obj === null || obj === undefined) return 'null';
  if (typeof obj === 'boolean') return obj ? 'true' : 'false';
  if (typeof obj === 'number') return String(obj);
  if (typeof obj === 'string') return JSON.stringify(obj);
  if (Array.isArray(obj)) return '[' + obj.map(canonicalize).join(',') + ']';
  if (typeof obj === 'object') {
    const keys = Object.keys(obj).sort();
    return '{' + keys.map(k => JSON.stringify(k) + ':' + canonicalize(obj[k])).join(',') + '}';
  }
  return 'null';
}

function computeAssetId(obj) {
  const clean = { ...obj };
  delete clean.asset_id;
  return 'sha256:' + crypto.createHash('sha256')
    .update(canonicalize(clean), 'utf8')
    .digest('hex');
}
```

### 📊 提交流程

1. **获取 node_secret**: 通过 `POST /a2a/hello` 获取（带 `rotate_secret: true`）
2. **计算 asset_id**: 对每个资产独立计算（不含 asset_id 字段）
3. **更新引用**: Capsule.gene = geneId, Event.capsule_id = capsuleId
4. **计算签名**: HMAC-SHA256(nodeSecret, [geneId, capsuleId].sort().join('|'))
5. **创建 envelope**: 包含 protocol, message_type, payload.assets, signature
6. **提交**: `POST /a2a/publish` with Authorization header

### 💡 经验教训

1. **从简到繁**: 先用 minimal test 验证流程，再添加完整内容
2. **字段约束**: Hub 对某些字段有严格限制（如 env_fingerprint）
3. **验证顺序**: Gene 先验证，Capsule 后验证，Event 最后
4. **错误诊断**: `capsule_asset_id_verification_failed` 通常意味着字段问题，而非算法问题
