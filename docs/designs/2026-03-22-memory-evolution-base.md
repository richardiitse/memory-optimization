## Vision

### 10x Check
Agent 记忆系统不只是存储——是真正的进化。用户说"每天都要叫 agent 回忆"，解决方案不是更好的检索，而是让 agent 形成肌肉记忆。类比：不是更好的笔记应用，而是第二个大脑。

### Platonic Ideal
记忆进化基座让 agent 从"每次像第一次见面"进化到"像一个越来越了解你的老朋友"。第三天重复任务时 agent 自动用之前成功的方式执行，不需要提醒。记忆有生命：会生长、会衰减、会遗忘、会 Consolidate 成真正的智慧。

## Key Engineering Decisions

### 1. PreferenceEngine: LLM 判断 + 缓存机制
- 用 LLM 判断两次任务是否"实质相同"
- 缓存判断结果（N 分钟 TTL）避免重复调用

### 2. Consolidation: 保守策略
- 冲突 → 标记待审核，不自动合并
- Memory Poisoning 防护优先

### 3. Active Recovery: 混合策略
- Stage 1 启动时加载（核心身份）
- Stage 2/3 按需加载（context 发展时）

### 4. Working Memory: 文件接口
- 写入 `~/.openclaw/workspace/memory/working_memory.jsonl`
- 与现有 graph.jsonl 模式一致

### 5. Decay Engine: 混合触发
- 访问时实时 decay
- 每周完整性检查处理长期未访问实体

### 6. Confidence 转换: 提取辅助函数
- `_confidence_to_string()` 消除重复代码

### 7. LLMClient 异常: 区分类型
- Timeout/429 → 指数退避重试
- 其他异常 → 抛出

### 8. Rate Limit 测试: 完整测试
- Mock requests 模拟 429/timeout
- 验证指数退避行为

### 9. PreferenceEngine 测试: Mock LLM
- 验证三种情况：same/different/cache_hit

### 10. 索引文件: 按需生成
- 性能优化，不增加写入复杂度

## Success Criteria

### 基础标准
1. ✅ 用户第一次使用时零配置
2. ✅ 第三天重复任务时，agent 自动用之前方式执行
3. ✅ 用户说"这样做不对"后，系统更新技能而非继续错误
4. ✅ 支持跨平台（OpenClaw、Claude Code 等）
5. ✅ 本地运行，隐私优先

### 增强标准
6. ✅ 记忆不会无限增长 — Decay 机制确保低价值记忆自动衰减
7. ✅ 记忆不被污染 — provenance + validation 防止 Poisoning
8. ✅ Context window 不溢出 — Working Memory 分层压缩
9. ✅ Agent 主动推断而非被动查询 — 启动时 staged loading + 主动插入偏好
10. ✅ Consolidation 形成真正的 Semantic Memory — 不是存储，是归纳

## Error & Rescue Registry

| METHOD/CODEPATH | WHAT CAN GO WRONG | EXCEPTION CLASS | HANDLING |
|-----------------|-------------------|-----------------|----------|
| EntityExtractor.extract() | LLM API 超时 | openai.APITimeoutError | 指数退避重试 (max 3) |
| | LLM API 返回 429 | RateLimitError | 指数退避重试 (max 3) |
| | LLM 返回非 JSON | JSONDecodeError | 跳过，记录警告 |
| | LLM 返回空响应 | ValueError | 跳过 |
| memory_ontology.create_entity() | graph.jsonl 被占用 | PermissionError | 重试 3次 |
| | graph.jsonl 损坏 | json.JSONDecodeError | 验证，备份恢复 |
| PreferenceEngine.extract() | 无足够历史数据 | ValueError | 降低置信度 |
| ConsolidationEngine.consolidate() | 无重复 Episode | ValueError | 跳过 |
| | 冲突检测失败 | AssertionError | 标记待审核 |
| DecayEngine.apply_decay() | strength 已为 0 | ValueError | 跳过 |
| Dashboard.render() | KG 为空 | ValueError | 显示空状态 |
| | 实体数量超限 | MemoryError | 采样 Top 20 |

## Failure Modes

| 场景 | 应对 |
|------|------|
| graph.jsonl 损坏 | memory_ontology.validate() 检测，备份恢复 |
| LLM API 超时 (Consolidation) | 重试 3 次，标记实体待处理 |
| Dashboard 渲染大量实体 | 分页 + 采样，默认显示 Top 20 |
| 并发写入 graph.jsonl | fcntl.flock() 文件锁 |
| LLM prompt injection | Pydantic validation + JSON schema 校验 |

## Security

| 威胁 | 缓解措施 |
|------|---------|
| LLM Prompt Injection | Pydantic validation，JSON schema 校验 |
| graph.jsonl 篡改 | 文件权限 600，用户组隔离 |
| Session 伪造 | Session ID 由系统生成，不可控 |
| API Key 泄露 | 环境变量，不在代码中 |
